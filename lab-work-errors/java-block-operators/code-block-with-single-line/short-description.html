Читая любой учебник по JavaSE, вы узнали, что если тело блочного оператора (например, if, while или for) состоит только из одной строки, то нет необходимости заключать это тело в фигурные скобки.

Другими словами, следующий участок кода: 

    
	if (result == null) {
		return;
	}

с применением этого правила можно записать вот так: 

    
	if (result == null)
		return;

что, с первого взгляда, позволяет сократить ваш исходник на одну строку.

С технической или синтаксической точки зрения, все вышесказанное абсолютно верно, однако на практике такой подход считается ошибочным и не применяется. В реальных проектах вместо того, чтобы помнить об этом синтаксическом правиле, и каждый раз задаваться вопросами вроде:

А сколько там уже строк в теле моего очередного блока?

или

А не пора ли мне уже заключать этот блок в фигурные скобки?

используют куда более простое и категоричное правило из Java Code Conventions:

ВЫ ОБЯЗАНЫ ПОМЕЩАТЬ ЛЮБОЙ БЛОК КОДА МЕЖДУ ОТКРЫВАЮЩЕЙ И ЗАКРЫВАЮЩЕЙ ФИГУРНЫМИ СКОБКАМИ, НЕЗАВИСИМО ОТ ТОГО, СКОЛЬКО СТРОК ВХОДИТ В ЭТОТ БЛОК!

Следовательно, для исправления текущей ошибки, вы должны пересмотреть в своей программе все блоки кода (в циклах, условиях и так далее), и проверить, стоят ли вокруг этих блоков фигурные скобки.
