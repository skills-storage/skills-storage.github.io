Скорее всего, из заголовка этой ошибки вы не поняли ничего, поэтому мы сразу приведем пример. Пусть вам необходимо написать метод, который принимает на вход один параметр целого типа, и определяет, является этот параметр положительным числом или отрицательным.

То есть, вы будете возвращать true, если принимаемый параметр больше нуля и false — в противном случае.

Тогда метод может называться isPositive(), и код его будет выглядеть следующим образом: 

    
	public boolean isPositive(int value) {
		if (value > 0) {
			return true;
		} else {
			return false;
		}
	}

Собственно, этим кодом мы уже и обозначили ошибку. Посмотрите, для того чтобы написать этот метод вам потребовалось 5 строк и один условный оператор If-else, хотя на самом деле тело метода может (и должно!) состоять только из одной строки.

Поскольку есть опасность, что я снова начну рассказывать о том, как космические корабли бороздят, лучше сразу привести правильный вариант такого метода: 

    
	public boolean isPositive(int value) {
		return (value > 0);
	}

Ну а теперь давайте порассуждаем — в чём же здесь проблема? Мы помним, что:

Чем меньше кода, тем лучше

Поэтому рассуждать о том, почему второй вариант лучше — мы не будем. Вместо этого мы задумаемся о том

Какого чёрта второй вариант работает??!!

А для того чтобы это сделать нам сначала необходимо прочитать первый вариант вашего кода. Давайте ещё раз его повторим — ну мало ли, кто-нибудь забыл: 

    
	public boolean isPositive(int value) {
		if (value > 0) {
			return true;
		} else {
			return false;
		}
	}

Теперь давайте попытаемся прочитать этот вариант вслух, но не jav-овски, а по-русски. У нас получится примерно следующий текст:

Если value больше нуля, то выражение (value > 0) возвращает мне true, и тогда условие в скобках оператора if получает результат true, поэтому я захожу в первый блок кода оператора if-else и тоже возвращаю true.

В противном случае выражение (value > 0) возвращает мне false, поэтому я захожу во второй блок оператора if-else и тоже возвращаю false.

Длинновато, правда? Но посмотрите, в этом русском тексте мы два раза повторили одну и ту же фразу — ... выражение (value > 0) возвращает.... И вот прямо сейчас нам нужно остановиться и закончить этот раздел упоминанием одного простого правила! Внимание:

Операторы логического сравнения, такие как > < >= <= == != работают с двумя операндами, и возвращают boolean результат

То есть эти операторы уже сами по себе возвращают либо true либо false. Зачем же тогда вы дублируете возврат этого результата своих блоках if-else? Просто возвращайте наружу тот результат, который вам отдают операторы сравнения!

И тогда мы ещё запишем наш изменённый метод:

    
	public boolean isPositive(int value) {
		return (value > 0);
	}

И попытаемся прочитать по-русски уже его:

Я проверяю, является ли value больше нуля, и возвращаю результат этой проверки.

Вот и всё

Однако если позволите ещё пару слов, почему появилось описание этой ошибки. Ведь, как вы понимаете, никакой опасности код с if-else сам по себе не таит.

Конечно, можно придумать ситуацию, когда кто-то в условии if поменяет больше на меньше, а return в блоке if не изменит, и тогда ваш метод будет работать неправильно так как он будет работать наоборот, однако такой ход мыслей приемлем скорее для книги, чем для реальной жизни.

Найти человека, который будет отбалды менять логические операторы в условиях, не проверяя за собой код, да и вообще не отдавая себе отчета — нафига он это делает — очень-очень сложно. Как правило, если таких людей находят, то их сразу же увольняют, поэтому раздолбай такого уровня — большой дефицит в любой компании
