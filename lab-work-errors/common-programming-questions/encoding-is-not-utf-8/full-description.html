Тут, друзья стоит еще раз повторить несколько тезисов:

  1. Однобайтовые кодировки не способны закодировать все национальные языки нашей планеты
  2. Поэтому в однобайтовых кодировках национальные языки всегда кодируются по-разному
  3. Символы национальных алфавитов, закодированные одной однобайтовой кодировкой будут некорректно отображаться в другой кодировке
  4. 2-х байтовые кодировки способны корректно закодировать все символы всех национальных языков нашей планеты
  5. UTF-8 — 2-х байтовые кодировка
  6. Любой символ национального алфавита, закодированный в UTF-8 в одном редакторе, корректно отобразится в другом редакторе на другом компьютере, при условии, что оба редактора настроены на кодировку UTF-8
  7. Тип char в Java имеет размер в 2 байта
  8. Значит Java по умолчанию использует 2-х байтовую кодировку UTF-8
  9. Значит в Java никогда не будет проблем с кодированием символов национальных алфавитов
  10. Если только вы не станете...

Намеренно кодировать их в однобайтовых кодировках!

Примечание-1:

Когда вы выучитесь и станете супер-классными программистами, вы можете вернуться к этой статье, и увидеть, что она насквозь пронизана ложью. Например:

  • UTF-8 занимает не 2 байта, а от 1-го до 4-х.
  • Java хранит строки не в UTF-8, а в UTF-16
  • Начиная с Java-9, появились Compact Strings, и строка, состоящая только из латиницы, будет хранить не массив символов, а массив байт. То есть, каждый символ такой строки будет посещаться в один байт, а не в два.

И, наверное, вы найдете еще много чего неправильного в этой статье. Но все это будет потом, когда вы станете супер-классными программистами.
