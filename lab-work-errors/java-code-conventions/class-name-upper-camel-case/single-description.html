Имена всех классов в вашей Java программе должны быть записаны в стиле UpperCamelCase (ВерхнийВерблюжийРегистр, блин ).

Такой стиль можно описать следующим правилом:

UpperCamelCaseТребуетЧтобыКаждаяПерваяБукваКаждогоСловаБылаЗаглавной

Больше здесь и сказать-то нечего, но давайте попробуем эти правила систематизировать:

  1. Еще раз — Каждая Первая Буква Каждого Слова Должна Быть Заглавной .
  2. Все остальные буквы в слове — маленькие (строчные).
  3. Если имя класса состоит только из одного слова, то у него только одна (первая) заглавная буква.
  4. Если имя класса состоит из нескольких слов, то между словами не используются никакие разделители! Разделителем уже является та самая первая заглавная буква! То есть, слова как-будто сцепляются друг с другом.

Примечание:

Для правила Если имя класса состоит только из одного слова, то у него только одна (первая) заглавная буква существуют исключения, которые касаются аббревиатур.

Если прямо сейчас вам жуть как не терпится узнать, что же там такого исключительного в аббревиатурах, то можете перейти к ошибке Аббревиатуры в именах классов

И напоследок приведем несколько примеров неправильного написания имен классов, и как их сделать правильными.

1. Имена классов из одного слова.

2. Имена классов из двух и более слов.

3. Неправильные разделители в именах классов.
