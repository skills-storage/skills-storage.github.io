Тезисы: Когда вы сами генерируете AUTO INCREMENT ID, вы либо: 1. Увеличиваете длину своих DML-запросов Зацените, вот так без ID - озвучь количество символов А вот так с - ID - снова укажи количество символов и скажи, насколько больше получилось. 2. Либо - "крадете" ID-шки. Просто отсылая число большее, чем на самом деле сейчас должно быть. (дай пример). Но все это херня, потому что: Вы задали AUTO INCREMENT, чтобы база данных генерировала вам айдишки AUTO INCREMENT есть свои недостатки Ссылку на недостатки Но у AUTO INCREMENT есть одно неоспоримое достоинство - вы сделали так, чтобы не ебать себе мозги со значениями для ID! Внимание вопрос - если вы так сделали, какого же хера вы теперь пытаетесь выебать мозги СУБД?
