GitHub — это, реально, клёвый ресурс. А выбор правильной почты для своих клёвых ресурсов — это, реально, ответственный шаг. И когда два эти компонента — клёвость и ответственность — сходятся вместе, сразу же возникает желание написать сто тыщ абзацев пафосного текста о том, как это клёво и ответственно!

А это опасно тем, что мы не только забудем, зачем нам нужно создавать GitHub аккаунт, но и

Вообще забудем, чем именно мы сейчас занимаемся!

Поэтому, вместо сто тыщ абзацев пафосного текста, мы ограничимся лишь пафосными тезисами:

  1. GitHub — это не просто сайт!
  2. На GitHub-е вы не регистрируетесь между делом в метро, чтобы получить промокод!
  3. GitHub — это ваша главная социальная сеть!
  4. GitHub — это ваша будущая карьера!
  5. GitHub — это сайт, где вы покажете свой код!
  6. Поэтому:

«Не нанимай на это дело идиотов!» (большой куш в видео а здесь картинка)

Что, в итоге, приведет нас к еще более пафосному выводу:

ИСПОЛЬЗУЙТЕ ДЛЯ СВОЕГО GITHUB АККАУНТА ВАШ НАСТОЯЩИЙ РАБОЧИЙ EMAIL!

И вот здесь — снова вызов на сто тыщ абзацев пафосного текста! Ведь, действительно, как это:

Создать правильный рабочий email?

Но, слава Богу, у нас для этого уже есть отдельное упражнение. И если вы сейчас здесь, то вы, скорее всего, его уже проходили.

А если не проходили, то и не надо его проходить. Давайте снова ограничимся тезисами:

  1. Не регистрируйте свою почту на местных или сомнительных аккаунтах!
  2. Вместо этого используйте международную компанию! Лучше всего — Gmail
  3. Лучшим именем для вашего почтового ящика будет комбинация вашего имени и фамилии.

Поэтому вывод здесь примерно такой (я использую себя в качестве примера):

Если я создам себе почту mega-pihar@yahoo.eu

То я сделаю неправильный выбор!

А если мне удастся отхватить mlazarevich@gmsil.com

То всё будет хорошо!

Перепиши здесь нахер все
